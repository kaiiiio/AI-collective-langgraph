# Slide 1: Reliable AI Workflows

## Concept
Open by saying that AI reliability is mostly workflow reliability. Tell students the model is only one part of the system; the real lesson is how state, nodes, routes, and traces keep the work understandable.

## How to Explain It
Use the inductive rhythm: name the concept, connect it to the real project example, point at the code or flow, then ask students to identify the state change.

## Student Check
Ask students to name one AI task they have used where the output was hard to debug.
