# Slide 8: Content State

## Concept
Explain that state is the contract between nodes. It carries request ids, context, prompts, raw model output, parsed content, errors, retry count, and final content.

## How to Explain It
Use the inductive rhythm: name the concept, connect it to the real project example, point at the code or flow, then ask students to identify the state change.

## Student Check
Ask which fields exist before the model call and which exist after it.
