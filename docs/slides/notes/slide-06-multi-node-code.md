# Slide 6: Multi-Node Code

## Concept
Show that multi-node graphs are just connected functions. Each node reads fields already created by previous nodes and returns one new update.

## How to Explain It
Use the inductive rhythm: name the concept, connect it to the real project example, point at the code or flow, then ask students to identify the state change.

## Student Check
Ask what would break if build_prompt ran before fetch_context.
