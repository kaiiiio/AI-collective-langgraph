# Slide 28: Production Monitoring

## Concept
Explain that LangSmith is useful after deployment too: failure rate, retries, latency, cost, and bad examples that become future eval data.

## How to Explain It
Use the inductive rhythm: name the concept, connect it to the real project example, point at the code or flow, then ask students to identify the state change.

## Student Check
Ask which metric would tell you the workflow is getting worse.
