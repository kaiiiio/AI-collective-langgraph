# Slide 26: LangSmith Debugging

## Concept
Connect LangSmith to concrete debugging questions from both projects: broken JSON, repeated failures, slow providers, missing evidence, and gap origins.

## How to Explain It
Use the inductive rhythm: name the concept, connect it to the real project example, point at the code or flow, then ask students to identify the state change.

## Student Check
Ask students which question they would check first after a production failure.
