# Slide 2: The Real Problem

## Concept
Frame the core pain: calling an LLM is easy, but controlling parsing, retries, validation, review, saving, and monitoring is where software engineering begins.

## How to Explain It
Use the inductive rhythm: name the concept, connect it to the real project example, point at the code or flow, then ask students to identify the state change.

## Student Check
Ask: if the model returns broken JSON, where should that failure live in code?
